# Contributing to Qairn

Thank you for contributing to Qairn.

## Development Workflow

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Commit clearly
5. Open a pull request

## Code Style

- Keep components modular
- Prefer reusable UI components
- Use TypeScript types where possible
- Keep backend logic inside services
- Avoid large controllers

## Commit Naming

Examples:
- feat: add realtime chat
- fix: resolve auth bug
- refactor: improve feed hooks

## Pull Requests

Before opening a PR:
- Ensure the app builds correctly
- Remove unused code
- Avoid committing secrets or .env files
- Test major flows before submitting

## Project Vision

Qairn aims to become a unified campus ecosystem platform for students.
